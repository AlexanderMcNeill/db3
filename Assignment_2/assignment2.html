  • customer ID

We are almost ready to carry out classification on the training data. However, there is one last detail we need to fix. If you open up the matrix X or X_train in Spyder, you will notice that among the 9 features present, there is a wide range of scales being used. Some columns have values in the range [0,1], while others (Income) oscillate in the range [1000,89000]. As I have explained during lectures, such a wide distribution of scales is not good for a learning algorithm. Hence, we need to normalize/transform the features to similar scales while maintaining the information contained in the data. This procedure is called feature scaling (bringing all the features to a similar range [0,1] for instance). Doing so is fairly straightforward in scikit–learn.

In [3]:
from sklearn import preprocessing
min_max_scaler = preprocessing.MinMaxScaler()
X_train_norm = min_max_scaler.fit_transform(X_train)
X_test_norm = min_max_scaler.fit_transform(X_test)
